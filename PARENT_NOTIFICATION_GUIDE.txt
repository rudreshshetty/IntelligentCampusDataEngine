================================================================================
        ABSENT STUDENT PARENT NOTIFICATION SYSTEM - COMPLETE GUIDE
================================================================================

OVERVIEW:
=========
The system automatically sends SMS notifications to parents/guardians when their
child is marked absent in a class. This is now fully implemented and enhanced 
with better validation and user feedback.

================================================================================
HOW IT WORKS:
================================================================================

1. MARKING ATTENDANCE:
   • Lecturer logs in to the dashboard
   • Navigates to "Attendance" section
   • Selects a date and subject for which to mark attendance
   • ONLY clicks on students who are ABSENT (optional)
   • When marked as ABSENT, a bell icon (🔔) appears as notification indicator
   • The toggle background changes to red to highlight absent status
   • ✚ All students NOT clicked (unmarked) are automatically marked PRESENT
   • ✚ No need to click every single student - much faster workflow!

2. SIMPLE WORKFLOW:
   • Only mark students who are ABSENT (click their toggle)
   • Leave all other students unmarked (they default to PRESENT)
   • This makes the process MUCH faster and more intuitive!
   • For a class with 100 students where 5 are absent:
     - OLD: Click all 100 toggles (95 Present + 5 Absent)
     - NEW: Click only 5 toggles (for the absent students)

3. SAVING & CONFIRMATION:
   • Click the "Save Attendance" button
   • If there are ABSENT students marked:
     - Confirmation dialog shows which parents will be notified
     - Shows list of absent students and their parent contact numbers
     - Option to cancel if needed
   • If all students are unmarked (all PRESENT):
     - Simple confirmation that all students will be marked PRESENT
   • After confirming, attendance is saved instantly

4. PARENT NOTIFICATION (AUTOMATIC):
   • Parents are ONLY notified if their child is marked ABSENT
   • All other students are automatically marked PRESENT (no notification sent)
   • Parents receive an SMS message with:
     - Student's name
     - Class/Subject name  
     - Date of absence
     - University contact information
   
   • Example SMS: "Hi, This is to inform that [STUDENT NAME] was absent from 
     the [SUBJECT] class on [DATE]. Please contact the university if you need 
     more information. - Sapthagiri NPS University"

5. INSTANT FEEDBACK:
   • After saving, a detailed modal shows:
     - ✓ Attendance saved successfully
     - Statistics (Total, Present, Absent)
     - Note that unmarked students were automatically marked PRESENT
     - List of parents who received notifications with contact numbers
     - All records added to attendance history

================================================================================
REQUIRED SETUP:
================================================================================

OPTION A: WITH TWILIO (RECOMMENDED FOR PRODUCTION):
------------------------------------------------------
To actually send SMS messages via Twilio, set these environment variables:

   1. Get Twilio credentials from https://www.twilio.com
   2. Set environment variables in your system or .env file:

      TWILIO_ACCOUNT_SID=your_account_sid
      TWILIO_AUTH_TOKEN=your_auth_token
      TWILIO_PHONE_NUMBER=your_twilio_phone_number
      MONGODB_URI=mongodb://localhost:27017/

   3. Ensure the `twilio` Python package is installed:
      pip install twilio

   4. Parent phone numbers must be in the student database
      (already stored in the students collection)

OPTION B: WITHOUT TWILIO (DEVELOPMENT/TESTING):
-------------------------------------------------
If Twilio is not configured, the system will:
   • Still log all SMS notifications to the database
   • Create SMS records for audit trail
   • Record them in the "sms_logs" collection
   • Show confirmation to the lecturer that notifications were processed

This is perfect for development and testing!

================================================================================
PARENT CONTACT INFORMATION:
================================================================================

Parent contact numbers MUST be stored in the student database:

Database Field: parent_contact (in students collection)

The system checks for parent contact in this priority:
1. parent_contact field (primary)
2. phone field (backup 1)
3. contact field (backup 2)

Make sure at least ONE of these fields contains the parent's phone number
for SMS notifications to work!

================================================================================
VIEWING NOTIFICATION HISTORY:
================================================================================

All notifications can be viewed in MongoDB:

1. SMS Logs Collection:
   db["sms_logs"].find()

2. Notifications Collection:
   db["notifications"].find()

3. Attendance Records (with parent notification flag):
   db["attendance"].find({"parent_notified": true})

Query to see all notifications for a specific date:
   db["notifications"].find({"date": "2026-03-01"})

Query to see all notifications for a specific student:
   db["notifications"].find({"enrollment_no": "25SUPMCA0001"})

================================================================================
TROUBLESHOOTING:
================================================================================

ISSUE: No SMS being sent even with Twilio configured
SOLUTION:
   1. Check environment variables are set (echo $TWILIO_ACCOUNT_SID)
   2. Verify Twilio account has credits/balance
   3. Check phone numbers include country code (e.g., +91 for India)
   4. Check sms_logs collection for error details

ISSUE: "Parent contact" showing as "N/A"
SOLUTION:
   1. Update student record with parent_contact field
   2. Use MongoDB update:
      db.students.updateOne(
        {enrollment_no: "25SUPMCA0001"},
        {$set: {parent_contact: "9876543210"}}
      )

ISSUE: Students can't mark attendance
SOLUTION:
   1. Check user role is "lecturer"
   2. Ensure all students are loaded in the table
   3. Check browser console for JavaScript errors

ISSUE: Confirmation dialog shows wrong parent contact
SOLUTION:
   1. Verify student record has correct parent_contact
   2. Check data is being passed correctly from template
   3. Update student database if contact has changed

================================================================================
API ENDPOINTS:
================================================================================

POST /save_bulk_attendance
   • Saves attendance for multiple students
   • Triggers SMS notifications automatically
   • Returns success with list of parents notified

Response format:
{
  "status": "success",
  "message": "Attendance saved successfully...",
  "present_count": 150,
  "absent_count": 5,
  "total_students": 155
}

Absence records are stored in:
   • attendance collection (main attendance record)
   • notifications collection (notification details)
   • sms_logs collection (SMS delivery logs)
